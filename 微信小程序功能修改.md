根据本文件夹中的front和end文件夹的前后端代码，完成pmf_mini_program小程序代码的修改：

A.背景

- 项目由三部分组成：`front`（Web前端）、`end`（后端）、`pmf_mini_program`（小程序）。
- 本次修改**仅针对** `pmf_mini_program` 文件夹。
- 所有新增或修改的功能，在 `front` 和 `end` 中已有完整实现，请完全参照其逻辑和样式。



B. 通用约束

1. **角色权限**：代码中需根据用户角色（`管理部门` / `非管理部门`）控制功能入口和按钮显示。
2. **接口对接**：新增功能需调用后端 `end` 中已有的 API，若涉及新接口，请参照 Web 端 `front` 的调用方式。
3. **UI 风格**：保持小程序现有风格，图表适配移动端屏幕，必要时允许横向滑动。



C.详细修改点

一、首页

1. 首页-快捷功能栏目内的内容交换顺序，从上到下，从左到右，依次是：

   - 批次管理/	入库管理/	出库管理/	货物管理

   - 质检管理/	数据看板/	温湿度上传/	个人信息

   （前五个是已有的，后三个是新增的）

2. 对于批次管理，仅管理部门的用户点击进入界面后，能新增批次（即已有的右上角的“+”号），对非管理部门的用户界面，隐藏该＋号
3. 入库管理没有修改
4. 出库管理没有修改
5. 货物管理没有修改
6. 质检管理没有修改
7. 对于数据看板，仿照 Web 端（`front`）图表，完成移动端界面的搭建，从上到下依次是：出入库趋势（曲线图）、货物库存数量（条形图）、仓库温湿度记录（条形图）、仓库容量饱和度（条形图，横向），可以参考前端的图表样式，但大小要适应移动端屏幕，若图表过宽则支持横向滑动，数据来源参照 Web 端接口
8. 对于温湿度上传，仿照front和end后端的内容，仅非管理部门的用户能点击进入设置界面，设置界面的内容和逻辑和网页端完全相同，管理部门的用户点击后，无法进入，同时弹出弹窗显示“仅质检员和温度检测员可上传温湿度”（和网页端一样）
9. 对于个人信息，点击后直接跳转到“我的-个人信息”界面（已有页面，只需配置跳转）

二、库存

此界面不用修改

   - 改为直接弹出**弹窗**，内容包含：
     - “XX单详情”（如入库单详情）

   - 弹窗标题分别为“库存盘点单详情”、“库存调整单详情”，并包含对应明细列表
