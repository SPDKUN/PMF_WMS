# 测试计划与测试用例说明——预制菜WMS仓储管理系统

## 1. 测试概述

### 1.1 测试目标

- 验证所有核心业务流程（入库、出库、质检、盘点、调整、次品处理）的功能正确性
- 验证32个控制器142个API接口的响应正确性
- 验证前端8个页面+微信小程序18个页面的UI交互正确性
- 验证数据库22张表的完整性和事务一致性
- 验证JWT认证、权限控制、AES加密的安全性

### 1.2 测试范围

| 子系统 | 测试内容 | 优先级 |
|--------|----------|--------|
| 后端API | 142个接口的功能测试 | P0 |
| 核心业务流程 | 入库、出库、质检、盘点、调整、次品处理 | P0 |
| 前端Web端 | 8个页面的UI交互 | P1 |
| 微信小程序 | 18个页面的功能验证 | P1 |
| 认证安全 | JWT登录、权限校验、密码加密 | P0 |
| 数据看板 | 4个图表的API数据准确性 | P2 |
| AI助手 | DeepSeek对话与数据注入 | P2 |
| 跨平台 | Web端+小程序数据同步 | P2 |

### 1.3 测试环境

| 环境 | 配置 |
|------|------|
| 后端服务器 | Spring Boot 3, Java 17, localhost:8088 |
| 数据库 | MySQL 8.0, Redis 6, localhost |
| Web前端 | Vite Dev Server, Chrome/Edge浏览器, 1920×1080 |
| 微信小程序 | 微信开发者工具 Stable 1.06+, SDK 3.16.2 |
| API测试工具 | Postman / Swagger UI |

---

## 2. 测试策略

### 2.1 测试方法

| 测试类型 | 方法 | 覆盖范围 |
|----------|------|----------|
| 单元测试 | JUnit 5 + SpringBootTest (PmfApplicationTests) | Spring上下文加载验证 |
| 接口测试 | Swagger UI + Postman手动测试 | 142个API端点 |
| 功能测试 | 手工黑盒测试 (按测试用例执行) | 6大核心业务流程 |
| UI测试 | 手工页面验证 + 小程序真机调试 | Web端8页面 + 小程序18页面 |
| 安全测试 | 手工验证JWT/权限/加密 | 认证+授权+传输加密 |

### 2.2 测试优先级定义

| 级别 | 定义 | 说明 |
|------|------|------|
| P0 | 核心功能 | 必须通过，否则系统不可用 |
| P1 | 重要功能 | 应该通过，影响主要用户体验 |
| P2 | 一般功能 | 建议通过，影响辅助功能 |
