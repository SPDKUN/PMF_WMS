## 2. 数据库配置

### 2.1 连接配置 (application.yaml)

```yaml
spring:
  datasource:
    url: jdbc:mysql://127.0.0.1:3306/pmf?useSSL=false&serverTimezone=Asia/Shanghai&characterEncoding=utf8&allowPublicKeyRetrieval=true
    driver-class-name: com.mysql.cj.jdbc.Driver
    username: root
    password: 123456
  data:
    redis:
      host: localhost
      port: 6379

mybatis:
  mapper-locations: classpath:com/neuedu/pmf/mapper/*.xml
  type-aliases-package: com.neuedu.pmf.entity
  configuration:
    map-underscore-to-camel-case: true
```

### 2.2 后端实体位置

- Java实体: `backend/PMF/src/main/java/com/neuedu/pmf/entity/*.java`
- Mapper接口: `backend/PMF/src/main/java/com/neuedu/pmf/mapper/*.java`
- Mapper XML: `backend/PMF/src/main/java/com/neuedu/pmf/mapper/*.xml`
- 建表SQL: `backend/PMF/sql/create_all_tables.sql`

| 字段名 | 数据类型 | 长度 | 必填 | 默认值 | 说明 |

- 大仓库: 16区 × 8位 = 128库位
- 中仓库: 8区 × 8位 = 64库位
- 小仓库: 4区 × 8位 = 32库位

## 6. 关键业务规则

### 6.1 编号生成规则

所有单据编号格式：`前缀 + yyyyMMdd + 3位流水号`

| 单据类型 | 前缀 | 示例 |
|---------|------|------|
| 批次 | B | B20260711001 |
| 入库单 | I | I20260711001 |
| 出库单 | O | O20260711001 |
| 质检单 | C | C20260711001 |
| 盘点单 | P | P20260711001 |
| 调整单 | A | A20260711001 |
| 次品处理单 | D | D20260711001 |

实现方式：每个Mapper都有`findTodayXxxNos`方法，查询当日已有编号，取最大流水号+1。

### 6.2 库存状态流转

```
创建批次 → inventory.status = '待入库'
  ↓ (完成入库)
inventory.status = '正常'
  ↓ (创建出库, 锁定)
inventory.locked_quantity += n
  ↓ (完成出库)
inventory.quantity -= n; inventory.locked_quantity -= n
  ↓ (quantity归零)
inventory记录删除
```

### 6.3 批次状态流转

```
创建批次 → batch.status = '待检'
  ↓ (完成质检)
batch.status = '正常'
  ↓ (质检不合格)
batch.status = '报废' → inventory.status = '待报废'
  ↓ (次品处理完成)
级联删除: batch → QC详情 → 盘点详情 → 出库详情 → inventory → batch
```

### 6.4 库位分配算法

入库时库位分配逻辑：
1. 确定货物储存条件 → 映射到仓库类型(常温→一般仓库, 冷冻→冷冻库, 恒温→恒温库)
2. 筛选匹配仓库类型下的未锁定空闲库位
3. 按库区分组展示给用户选择
4. 分配规则：每个库位放10单位，剩余放最后库位
5. `所需库位数 = ceil(初始数量 / 10)`
