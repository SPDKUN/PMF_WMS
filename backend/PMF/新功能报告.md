# PMF后端新功能添加报告

## 概述

本次更新参考了 `F:\2026summer\教程\2026-7-8\ems` 项目，为 PMF 后端添加了以下新功能：JWT登录认证、Excel导出、Redis缓存会话。

---

## 一、新增文件

### 1. AuthController — 登录认证控制器
**文件：** `controller/AuthController.java`  
**接口：** `GET /auth/login?username=xxx&password=xxx`

**功能说明：** 独立的认证控制器，专门处理用户登录请求。接收用户名和密码，调用 AuthService 进行验证。登录成功后调用 JwtUtil 生成 JWT Token 并返回给前端，前端后续请求携带此 Token 即可通过 JWT 过滤器验证。

**原理：** 将认证逻辑从 UserController 分离出来（单一职责原则），Controller 层只负责接收参数和返回结果，不包含业务逻辑。

---

### 2. AuthService / AuthServiceImpl — 认证业务层
**文件：** `service/AuthService.java`、`service/impl/AuthServiceImpl.java`

**功能说明：** 封装登录业务逻辑。AuthServiceImpl 调用 UserMapper 查询数据库验证用户名密码，登录成功后将用户信息存入 Redis（key 格式：`username_xxx`），设置30分钟过期。

**原理：**
- **数据库验证：** 通过 `findByUsernameAndPassword` 查询 user 表匹配用户
- **Redis缓存会话：** 登录成功后，将用户对象以 JSON 形式存入 Redis，用于后续快速获取当前用户信息或实现会话管理，避免每次操作都查数据库
- **分层解耦：** AuthService 专注于认证，与普通的 UserService CRUD 操作分离

---

### 3. HelloController — 测试端点
**文件：** `controller/HelloController.java`  
**接口：** `GET /h`

**功能说明：** 简单的健康检查/测试端点，返回字符串 "hello"。常用于验证服务是否正常启动、负载均衡健康检查等场景。

## 三、新功能原理总结

```
用户登录流程：
  前端 POST /auth/login  ──→  AuthController
                                  │
                             AuthService.login()
                                  │
                        ┌─────────┴─────────┐
                        │                   │
                   UserMapper           RedisUtil
                   (查数据库)          (缓存会话30分钟)
                        │                   │
                        └─────────┬─────────┘
                                  │
                             JwtUtil.generateToken()
                                  │
                            返回 Token 给前端

后续请求流程：
  前端(携带Token)  ──→  JwtAuthenticationFilter
                              │
                    验证Token + 提取userId
                              │
                        ┌─────┴─────┐
                      通过          拒绝
                        │            │
                      Controller   返回TOKEN_ERROR
```

本次添加的核心是 **JWT 无状态认证体系**：登录后签发 Token，后续请求携带 Token 即可验证身份，无需服务端存储 Session，适合分布式部署场景。同时结合 Redis 缓存用户会话信息，在无状态的 Token 机制上增加了有状态会话管理的能力。

### 4.2 JWT 配置常量 — JwtConstants.java

```java
public class JwtConstants {
    // 签名密钥（HMAC-SHA384 算法），生产环境应改为环境变量或配置中心管理
    public static final String JWT_SECRET = "health-management-backend-2026-jwt-secret-key-123456";

    // Token 过期时间：7200 秒 = 2 小时
    public static final long JWT_EXPIRE_TIME = 7200L;

    // 前端携带 Token 的 HTTP 请求头名称
    public static final String JWT_HEADER = "Authorization";

    // Token 前缀，标准格式：Bearer <token>
    public static final String JWT_PREFIX = "Bearer ";
}
```

**设计要点：**
- 使用 HMAC-SHA384 签名算法，需要密钥长度 ≥ 32 位
- Token 有效期 2 小时，过期后需重新登录获取新 Token
- 遵循 OAuth 2.0 规范，使用 `Authorization: Bearer <token>` 标准格式

### 4.3 JWT 工具类 — JwtUtil.java

#### 4.3.1 生成 Token — `generateToken()`

```java
public String generateToken(Long userId, String username) {
    SecretKey secretKey = Keys.hmacShaKeyFor(JwtConstants.JWT_SECRET.getBytes());

    Map<String, Object> claims = new HashMap<>();
    claims.put("userId", userId);    // 将用户ID存入Payload
    claims.put("username", username); // 将用户名存入Payload

    return Jwts.builder()
            .setClaims(claims)         // 自定义声明
            .setId(String.valueOf(userId)) // JWT ID
            .setIssuedAt(new Date())        // 签发时间
            .setExpiration(new Date(        // 过期时间 = 当前时间 + 7200秒
                System.currentTimeMillis() + JwtConstants.JWT_EXPIRE_TIME * 1000))
            .signWith(secretKey)            // 签名
            .compact();                     // 序列化为字符串
}
```

**流程解释：**
1. 使用 `JwtConstants.JWT_SECRET`（字符串密钥）创建 HMAC 签名密钥
2. 构建 Payload（负载），将 `userId` 和 `username` 写入自定义声明
3. 设置 JWT 标准字段：`jti`（JWT ID）、`iat`（签发时间）、`exp`（过期时间）
4. 使用密钥签名后压缩为形如 `eyJhbGciOiJIUzM4NCJ9.xxx.xxx` 的三段式字符串

**生成的 JWT 结构：**

```
Header（头部）:  {"alg": "HS384"}                              ← Base64编码
Payload（负载）: {"userId":1,"username":"admin","exp":...,"iat":...}  ← Base64编码
Signature（签名）: HMACSHA384(Header + "." + Payload, secret)  ← 防篡改
```

最终 Token 格式：`Header.Payload.Signature`（三段用 `.` 分隔）

#### 4.3.2 解析 Token — `parseToken()`

```java
public Claims parseToken(String token) throws Exception {
    SecretKey secretKey = Keys.hmacShaKeyFor(JwtConstants.JWT_SECRET.getBytes());
    return Jwts.parserBuilder()
            .setSigningKey(secretKey)  // 设置验签密钥
            .build()
            .parseClaimsJws(token)     // 解析并验证签名
            .getBody();                // 获取Payload
}
```

**流程解释：**
- 使用与生成时相同的密钥构建解析器
- `parseClaimsJws(token)` 会同时验证签名是否正确、Token 是否过期
- 验证通过后返回 Claims 对象（即 Payload 内容）
- 签名不匹配或过期会抛出异常（如 `ExpiredJwtException`、`SignatureException`）

#### 4.3.3 验证 Token — `validateToken()`

```java
public boolean validateToken(String token) {
    try {
        Claims claims = parseToken(token);
        return !claims.getExpiration().before(new Date()); // 双重检查过期时间
    } catch (Exception e) {
        return false;  // 解析失败 = Token无效
    }
}
```

**流程解释：**
- 调用 `parseToken()` 尝试解析，解析成功即为签名有效
- 额外检查过期时间，双重保障
- 任何异常（过期、签名错误、格式错误）均返回 `false`

#### 4.3.4 从请求头提取 Token — `extractTokenFromHeader()`

```java
public String extractTokenFromHeader(String headerValue) {
    if (headerValue == null || !headerValue.startsWith(JwtConstants.JWT_PREFIX)) {
        return null;
    }
    return headerValue.substring(JwtConstants.JWT_PREFIX.length()).trim();
    // 去掉 "Bearer " 前缀（7个字符），得到纯Token
}
```

**流程解释：**
- 检查请求头是否为空、是否以 `"Bearer "` 开头
- 截取掉前缀后返回纯 Token 字符串
- 例如输入 `"Bearer eyJhbGciOiJIUzM4..."`，返回 `"eyJhbGciOiJIUzM4..."`

#### 4.3.5 从 Token 中提取信息

```java
// 获取用户名
public String getUsernameFromToken(String token) throws Exception {
    Claims claims = parseToken(token);
    return claims.get("username", String.class);
}

// 获取用户ID
public Long getUserIdFromToken(String token) throws Exception {
    Claims claims = parseToken(token);
    return claims.get("userId", Long.class);
}
```

### 4.4 登录接口 — AuthController.java

```java
@RequestMapping("/auth")
@RestController
public class AuthController {
    @Autowired
    AuthService authService;
    @Autowired
    JwtUtil jwtUtil;

    @GetMapping("/login")
    public ResultData login(@RequestParam("username") String username,
                            @RequestParam("password") String password) {
        User user = authService.login(username, password);
        if (user != null) {
            // 登录成功：生成JWT Token
            String token = jwtUtil.generateToken(user.getUser_id().longValue(), username);
            ResultData resultData = ResultData.success(user);
            resultData.put("token", token);  // 将Token放入响应体
            return resultData;
        }
        // 登录失败：用户名或密码错误
        return ResultData.fail(ResultCode.USER_AND_PASSWORD_ERROR);
    }
}
```

**接口信息：**
- **请求方式：** `GET`
- **请求路径：** `/auth/login`
- **请求参数：** `username`（用户名）、`password`（密码）
- **白名单：** 该路径在 `JwtAuthenticationFilter.EXCLUDE_PATHS` 中，无需携带 Token

**执行流程：**
1. 接收前端传来的 `username` 和 `password`
2. 调用 `authService.login()` 进行身份验证
3. 验证成功 → 用 `user_id` 和 `username` 生成 JWT Token
4. 将用户信息通过 `ResultData.success(user)` 放入 `data` 字段
5. 将 Token 额外放入响应体的 `token` 字段
6. 返回统一格式的 JSON 响应

### 4.5 认证业务逻辑 — AuthServiceImpl.java

```java
@Service
public class AuthServiceImpl implements AuthService {
    @Autowired
    private UserMapper userMapper;
    @Autowired
    private RedisUtil redisUtil;

    @Override
    public User login(String username, String password) {
        // 第一步：查询数据库验证用户名密码
        User user = userMapper.findByUsernameAndPassword(username, password);
        if (user != null) {
            // 第二步：登录成功，将用户信息缓存到Redis
            String redisKey = "username_" + username;
            redisUtil.set(redisKey, user, 30, TimeUnit.MINUTES);
            return user;
        }
        return null;
    }
}
```

**执行流程：**
1. 调用 `UserMapper.findByUsernameAndPassword()` 查询数据库
   - SQL：`SELECT * FROM user WHERE user_name = #{user_name} AND password = #{password}`
2. 查找到匹配用户 → 将用户对象以 key `username_xxx` 存入 Redis，TTL 30 分钟
3. 未找到 → 返回 `null`，Controller 层返回错误码 `1001`（用户名或密码错误）

**注意：** 当前密码以明文形式存储和比较，生产环境应使用 BCrypt 等哈希算法加密存储。

### 4.6 JWT 认证过滤器 — JwtAuthenticationFilter.java

```java
@Component
public class JwtAuthenticationFilter extends OncePerRequestFilter {

    // 白名单路径（无需Token即可访问）
    private static final String[] EXCLUDE_PATHS = {
            "/auth/login",         // 登录接口
            "/register",           // 注册接口（预留）
            "/swagger-ui/**",      // Swagger UI
            "/v3/api-docs/**",     // API文档
            "/upload/**",          // 文件上传
            "/captcha/**",         // 验证码
            "/temp/**",            // 临时文件
            "/favicon.ico"         // 网站图标
    };

    @Override
    protected void doFilterInternal(HttpServletRequest request,
                                    HttpServletResponse response,
                                    FilterChain filterChain) {
        try {
            // 第1步：OPTIONS 预检请求直接放行（CORS需要）
            if ("OPTIONS".equalsIgnoreCase(request.getMethod())) {
                filterChain.doFilter(request, response);
                return;
            }

            // 第2步：白名单路径直接放行
            if (isExcludePath(request.getRequestURI())) {
                filterChain.doFilter(request, response);
                return;
            }

            // 第3步：从Authorization请求头提取Token
            String token = jwtUtil.extractTokenFromHeader(
                request.getHeader(JwtConstants.JWT_HEADER));

            // 第4步：Token为空 → 抛出异常
            if (!StringUtils.hasText(token)) {
                throw new JwtException("没有登录，请先登录");
            }

            // 第5步：验证Token有效性
            if (!jwtUtil.validateToken(token)) {
                throw new JwtException("token无效或者过期");
            }

            // 第6步：验证通过，将userId存入request供后续使用
            Long userId = jwtUtil.getUserIdFromToken(token);
            request.setAttribute("userId", userId);

            // 第7步：放行请求到Controller
            filterChain.doFilter(request, response);

        } catch (JwtException e) {
            // Token校验失败 → 返回JSON错误
            response.setContentType("application/json");
            response.setCharacterEncoding("UTF-8");
            ResultData resultData = ResultData.fail(ResultCode.TOKEN_ERROR);
            response.getWriter().write(objectMapper.writeValueAsString(resultData));
        }
    }
}
```

**过滤器执行流程图：**

```
请求到达
    │
    ├─ 是 OPTIONS 请求？ ──是──→ 直接放行（CORS预检）
    │        │否
    ├─ 在白名单中？ ──是──→ 直接放行（登录/文档/静态资源）
    │        │否
    ├─ 提取 Authorization 头
    │        │
    ├─ Token 为空？ ──是──→ 返回 TOKEN_ERROR（未登录）
    │        │否
    ├─ Token 有效？ ──否──→ 返回 TOKEN_ERROR（过期/无效）
    │        │是
    ├─ 提取 userId 存入 request.setAttribute("userId")
    │
    └─ 放行到 Controller 处理业务
```

**白名单路径匹配机制：**
```java
private boolean isExcludePath(String requestUri) {
    for (String excludePath : EXCLUDE_PATHS) {
        String path = excludePath.replace("/**", ".*"); // 通配符转正则
        if (requestUri.matches(path)) {
            return true;
        }
    }
    return false;
}
```
将 `/**` 替换为正则 `.*` 实现通配符匹配。例如 `/swagger-ui/**` 变成 `/swagger-ui/.*`，可匹配 `/swagger-ui/index.html` 等所有子路径。

#### 第二步：携带 Token 访问受保护接口

登录成功后，将返回的 `token` 值保存到前端（localStorage / sessionStorage / Cookie）。

后续所有请求需在 HTTP Header 中携带 Token：

```
Authorization: Bearer eyJhbGciOiJIUzM4NCJ9.eyJ1c2VySWQiOjEsInVzZXJuYW1lIjoiYWRtaW4i...
```

格式为：`Bearer` + 空格 + Token值

**示例 — 获取用户列表：**
```
GET http://localhost:8088/user/list
Header: Authorization: Bearer eyJhbGciOiJIUzM4NCJ9...
```

#### 第三步：Token 过期后重新获取

Token 有效期为 2 小时（7200 秒）。过期后访问受保护接口会收到 `code: 1002` 错误。

此时前端需要：
1. 引导用户重新访问 `/auth/login` 接口
2. 用新的用户名密码获取新的 Token
3. 替换本地存储的旧 Token

---

### 4.9 前端集成示例

#### JavaScript（axios）示例：

```javascript
// 1. 登录获取Token
async function login(username, password) {
    const response = await axios.get('/auth/login', {
        params: { username, password }
    });
    if (response.data.code === 200) {
        const token = response.data.token;
        localStorage.setItem('token', token);  // 保存Token
        return response.data.data;             // 返回用户信息
    } else {
        throw new Error(response.data.msg);
    }
}

// 2. 配置axios拦截器，自动在请求头携带Token
axios.interceptors.request.use(config => {
    const token = localStorage.getItem('token');
    if (token) {
        config.headers['Authorization'] = `Bearer ${token}`;
    }
    return config;
});

// 3. 处理Token过期
axios.interceptors.response.use(
    response => response,
    error => {
        if (error.response?.data?.code === 1002) {
            localStorage.removeItem('token');
            window.location.href = '/login';  // 跳转登录页
        }
        return Promise.reject(error);
    }
);

// 4. 访问受保护接口
async function getUserList() {
    const response = await axios.get('/user/list');
    return response.data.data;
}
```

### 4.10 安全注意事项

1. **密码明文存储：** 当前 `user` 表中 `password` 字段以明文存储，数据库泄露将直接暴露所有用户密码。建议使用 BCrypt 或 SCrypt 哈希算法对密码加密后再存储。

2. **登录接口使用 GET 请求：** 用户名和密码通过 URL 查询参数传递，会被记录在浏览器历史、服务器访问日志中。建议改为 POST 请求，将敏感信息放入请求体。

3. **JWT 密钥硬编码：** `JwtConstants.JWT_SECRET` 直接写在代码中。生产环境应通过环境变量或配置中心注入，并定期轮换密钥。

4. **Token 无主动失效机制：** Token 签发后，在过期前无法主动撤销。如需实现"强制下线"功能，可配合 Redis 维护黑名单或使用短有效期 + Refresh Token 机制。

5. **不使用 HTTPS 时 Token 明文传输：** JWT Token 本身不加密（仅签名），内容Base64编码后任何人可解码查看Payload。敏感信息不要存入 JWT Payload，且应使用 HTTPS 传输。
