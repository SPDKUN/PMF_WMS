### 技术说明

- 所有表主键采用 VARCHAR(32)，新增时由 Service 层自动生成 UUID（去除连字符）
- Mapper XML 的 UPDATE 语句使用动态 `<set>` + `<if>` 标签，支持部分字段更新
- 表名含 MySQL 保留字（如 `zone`、`batch`）在 XML 中使用反引号包裹
- 日期字段使用 `LocalDate`（DATE 类型）和 `LocalDateTime`（DATETIME 类型）
- 金额/小数使用 `BigDecimal`，布尔字段使用 `Boolean`
- 统一返回 `ResultData` 格式：`{ "code": 200, "msg": "操作成功", "data": ... }`

### 遵循的编码规范

- Entity 使用 Lombok `@Data` + `@AllArgsConstructor` + `@NoArgsConstructor`
- 字段名使用 snake_case 匹配数据库列名
- Controller 使用 form-based 参数绑定（与现有 User/Book 模块一致）
- RESTful 风格：GET 查询 / POST 新增 / PUT 修改 / DELETE 删除
- 接口路径：`/{资源名}/list`（列表）、`/{资源名}/{id}`（详情）、`/{资源名}/{id}`（删除）
