## 三、库存查询（最复杂的一个）

库存查询使用了特殊的接口：

```
GET /inventory/listWithDetails?batchId=xxx
```

这个接口返回的数据不是简单的数据库记录，而是**多表联查**的结果：

```json
{
  "inventory_id": 1,
  "goods_name": "冷冻鸡排",      // 来自 goods 表
  "batch_id": "B20260711003",   // 来自 batch 表
  "location_name": "A-01-01",   // 来自 location 表
  "quantity": 50,               // 来自 inventory 表
  "inventory_status": "正常"
}
```

**InventoryVO** 就是为此设计的：

```java
// vo/InventoryVO.java
public class InventoryVO {
    private Integer inventoryId;
    private String goodsName;      // 不是 goods_id，而是货物名称
    private String batchId;
    private String locationName;   // 不是 location_id，而是库位名称
    private Integer quantity;
    private String inventoryStatus;
}
```

这种"把ID换成名称方便人看"的操作，本质上是 SQL 的 **JOIN 联表查询**。

### 后端 SQL 大致是这样：

```sql
SELECT
  i.inventory_id,
  g.goods_name,        -- 从 goods 表查名字
  i.batch_id,
  l.location_name,     -- 从 location 表查名字
  i.quantity,
  i.inventory_status
FROM inventory i
LEFT JOIN goods g ON i.goods_id = g.goods_id
LEFT JOIN location l ON i.location_id = l.location_id
```

## 四、操作日志查询

操作日志记录了系统中**每一次操作**，相当于"黑匣子"。

日志表结构：
| 字段 | 说明 |
|------|------|
| log_id | 日志ID |
| operator_id | 操作人ID |
| operation_type | 操作类型（入库/出库/盘点/质检...） |
| operation_content | 操作描述 |
| operation_result | 操作结果（成功/失败） |
| operation_time | 操作时间 |

**日志是只读的**——系统自动记录，不允许删除或修改，这样才能保证可追溯性。

## 五、Excel 导出功能

### 前端实现

```javascript
// utils/request.js 中的 exportExcel 方法
async exportExcel(url, fileName = "导出文件.xlsx") {
  // 1. 发送 GET 请求，指定 responseType: "blob"（二进制数据）
  const res = await service.get(url, { responseType: "blob" })

  // 2. 把二进制数据转成 Blob 对象（Excel 文件格式）
  const blob = new Blob([res.data], {
    type: "application/vnd.openxmlformats-officedocument.spreadsheetml.sheet"
  })

  // 3. 创建一个临时下载链接，触发浏览器下载
  const downloadUrl = URL.createObjectURL(blob)
  const a = document.createElement("a")
  a.href = downloadUrl
  a.download = fileName
  a.click()               // 模拟点击

  // 4. 释放内存
  URL.revokeObjectURL(downloadUrl)
}
```

**关键点**：`responseType: "blob"` 让 axios **不把返回结果当 JSON 解析，而是按原始二进制数据处理**。因为 **Excel 文件不是 JSON 格式**。

### 后端实现

后端使用 **EasyExcel**（阿里巴巴开源库）生成 Excel 文件：

```java
@GetMapping("/excel")
public void exportExcel(HttpServletResponse response) {
    // 1. 查询数据
    List<User> userList = userMapper.list();

    // 2. 设置响应头（告诉浏览器这是 Excel 文件）
    response.setContentType("application/vnd.openxmlformats-officedocument.spreadsheetml.sheet");
    response.setHeader("Content-Disposition", "attachment; filename=users.xlsx");

    // 3. 用 EasyExcel 写出
    EasyExcel.write(response.getOutputStream(), User.class).sheet("人员列表").doWrite(userList);
}
```

## 六、筛选和搜索

操作日志视图支持按**日期范围**和**操作类型**筛选：

```javascript
// 前端发送带参数的请求
request.get('/operationLog/list', {
  startDate: '2026-07-01',
  endDate: '2026-07-15',
  operationType: '入库'
})
```

后端 MyBatis 的 XML 中，SQL 会动态拼接 WHERE 条件：

```xml
<select id="list" resultType="OperationLog">
    SELECT * FROM operation_log
    WHERE 1 = 1
    <if test="startDate != null">
        AND operation_time >= #{startDate}
    </if>
    <if test="endDate != null">
        AND operation_time <= #{endDate}
    </if>
    <if test="operationType != null">
        AND operation_type = #{operationType}
    </if>
    ORDER BY operation_time DESC
</select>
```

这是 MyBatis 的**动态 SQL** 特性。`<if test="...">` 标签在条件为 true 时才拼接对应的 SQL 片段。

## 七、你可能被问到的问题

**Q: 库存明细中显示的是货物名称而不是 ID，这是怎么实现的？**

A: 后端使用 SQL 的 LEFT JOIN 联表查询，把 inventory 表和 goods 表、location 表关联起来，把 goods_id 替换为 goods_name，把 location_id 替换为 location_name。返回的 VO 对象（InventoryVO）包含了这些可读的名称。

**Q: Excel 导出是怎么实现的？**

A: 前端以 `responseType: "blob"` 方式请求后端导出接口，后端用 EasyExcel 库生成 Excel 文件的二进制流，前端收到后创建一个临时的下载链接触发浏览器下载。

**Q: 操作日志是什么时候记录的？**

A: 每次执行入库、出库、盘点、质检等操作时，在 Service 层的事务中同步记录。日志的写入和业务操作在同一个数据库事务中，保证了**一致性**——业务操作成功日志才能写入，业务失败日志也不会错误写入。
