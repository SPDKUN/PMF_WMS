# 功能详解二：主页与数据看板

---

## 一、主页（Home）

### 页面位置
- 前端：`front/src/pages/home/index.vue`
- 路由：`/home`

### 主页显示什么？

主页是一个"驾驶舱"概览面板，显示三部分内容：

1. **统计卡片**：总库存数量、待办任务数、最后更新时间
2. **快捷AI入口**：可以直接输入问题跳到AI助手页面
3. **待办列表**：质检任务和盘点任务的提醒

### 数据从哪来？

主页的数据来自后端的一个专属接口：

```
GET /ai/homeStats?assigneeId=<当前用户ID>
```

这个接口在 `AiController` 中（虽然跟AI聊天无关），返回：

```json
{
  "totalInventory": 1520,     // 总库存数量
  "pendingTasks": 5,          // 待办任务数
  "lastUpdate": "14:30"       // 最后操作时间
}
```

### 后端如何统计这些数据？

```java
// AiServiceImpl.getHomeStats()
public Map<String, Object> getHomeStats(Integer assigneeId) {
    // 1. 总库存 = 所有"正常"和"待出库"状态的库存数量之和
    //    （排除"待入库"——还没进仓库的；排除"待报废"——质检不合格的）
    int totalInventory = 0;
    for (Inventory inv : inventoryMapper.list()) {
        if (!"待入库".equals(status) && !"待报废".equals(status)) {
            totalInventory += inv.getQuantity();
        }
    }

    // 2. 待办任务 = 分配给当前用户的未完成任务数
    int pendingTasks = workTaskMapper.findByAssigneeId(assigneeId).size();

    // 3. 最后更新时间 = 操作日志中最新一条的时间
     String lastUpdate = "--";
}
```

### 快捷AI入口的工作原理

用户在主页输入框输入问题，点击发送：
```javascript
// 不是直接调AI接口，而是跳转到AI页面，把问题通过URL参数传过去
this.$router.push({ name: 'AiChat', query: { q: this.aiQuestion } })
```

AI页面在初始化时会检查 URL 参数，如果有 `q` 就自动发送这个消息。

## 二、数据看板（Dashboard）

### 页面位置
- 前端：`front/src/pages/dashboard/index.vue`
- 路由：`/dashboard`
- **注意**：这是唯一使用**懒加载**的页面（`() => import(...)`）

### 什么是懒加载？

```javascript
// 普通加载（打包时会把Dashboard代码和主包打在一起）
import Dashboard from '@/pages/dashboard/index.vue'

// 懒加载（用户第一次访问 /dashboard 时才去下载这个页面的代码）
() => import('@/pages/dashboard/index.vue')
```

**为什么对 Dashboard 用懒加载？** 因为 ECharts 图表库体积比较大，如果首页就加载会拖慢整个应用。懒加载让用户只有在访问看板页面时才下载图表相关的代码。

### 四个图表

数据看板用了 **ECharts**（百度开源的图表库）展示了 4 个图表，数据来自 4 个后端接口：

| 图表 | 接口 | 展示内容 |
|------|------|---------|
| 近7天出入库趋势 | `GET /dashboard/weeklyInboundOutbound` | 折线图，蓝线=入库量，绿线=出库量 |
| 货物数量分布 | `GET /dashboard/goodsQuantity` | 柱状图，每种货物有多少件 |
| 仓库温湿度 | `GET /dashboard/warehouseTempHumidity` | 双Y轴图，左轴=温度，右轴=湿度 |
| 仓库容量饱和度 | `GET /dashboard/warehouseCapacity` | 柱状图，每个仓库已用/总共多少库位 |

### ECharts 图表是怎么画出来的？

以"出入库趋势"为例（简化版）：

```javascript
// 1. 在 template 中准备一个容器
<div ref="chart1" style="width:100%; height:300px;"></div>

// 2. 在 mounted 生命周期中初始化图表
mounted() {
    this.chart1 = echarts.init(this.$refs.chart1)
    this.loadWeeklyData()
}

// 3. 请求后端数据，配置图表
async loadWeeklyData() {
    const res = await request.get('/dashboard/weeklyInboundOutbound')
    // res.data = { dates: ['周一','周二',...], inbound: [10,20,...], outbound: [5,15,...] }

    this.chart1.setOption({
        xAxis: { data: res.data.dates },        // X轴：日期
        yAxis: {},                               // Y轴：数量
        series: [
            { name: '入库', type: 'line', data: res.data.inbound },
            { name: '出库', type: 'line', data: res.data.outbound }
        ]
    })
}
```

### 后端数据是如何聚合的？

DashboardServiceImpl 直接对原始数据表进行 SQL 聚合查询。例如"近7天出入库趋势"就是查询 `inbound_order_head` 和 `outbound_order_head` 表，按日期分组统计每天的数量。

## 三、你可能被问到的问题

**Q: 主页的统计数据是实时更新的吗？**

A: 每次打开主页都会重新请求后端接口，获取最新的统计数据。但如果你一直停留在主页，数据不会自动刷新。

**Q: Dashboard 为什么用懒加载？**

A: 因为 ECharts 库体积较大（~1MB），懒加载可以减小首屏加载时间，让用户打开网站更快。只有真正访问看板页面时才会去下载图表相关代码。

**Q: ECharts 图表的数据从哪里来？**

A: 从后端专门的 Dashboard 接口获取。后端通过对原始数据表（入库单、出库单、仓库、温湿度记录）做聚合查询，返回适合图表展示的统计数据。
