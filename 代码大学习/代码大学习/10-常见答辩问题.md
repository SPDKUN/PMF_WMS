# 常见答辩问题与回答指南

> 本章汇总了老师可能在答辩中问到的问题，并给出建议的回答话术。**回答技巧：不要背诵，理解逻辑后用自己的话说。**

---

## 一、项目概述类问题

### Q1：介绍一下你的项目

**建议回答**：
"我做的是一个**预制菜行业的仓库管理系统（WMS）**。主要功能包括用户登录注册、人员/仓库/货物的管理、出入库操作、库存盘点、质量检查、次品处理、温湿度监控、数据看板和 AI 智能助手。系统采用前后端分离架构，前端用 Vue 3 + Element Plus，后端用 Spring Boot + MyBatis，数据库用 MySQL，AI 助手接入了 DeepSeek 大模型。"

### Q2：你在这个项目中负责了什么？

**建议回答**：
"这是我独立完成的课程项目。我负责了从需求分析、数据库设计、前后端开发到最终部署的完整流程。在开发中借助了 AI 辅助编程工具来提升开发效率，但我掌握了系统的整体架构和每项技术的应用方式。"

### Q3：项目的难点在哪里？

**建议回答**：
"主要有三个难点：
1. **出入库的业务流程**比较复——一次入库操作涉及 7-9 张数据库表的联动更新，需要用事务来保证数据一致性；
2. **AI 助手的实现**需要把真实数据库数据注入到提示词中，让大模型能基于实际数据回答问题而不是编造；
3. **前后端联调**时需要处理跨域、JWT 令牌传递、Excel 文件导出（二进制流处理）等问题。"

## 二、技术架构类问题

### Q4：你用了什么技术栈？

**建议回答**：

| 层级 | 技术 | 作用 |
|------|------|------|
| 前端框架 | Vue 3 | 构建用户界面 |
| UI 组件库 | Element Plus | 按钮、表格、弹窗等现成组件 |
| 图表 | ECharts | 数据看板的图表展示 |
| HTTP 请求 | Axios | 前后端通信 |
| 后端框架 | Spring Boot 3 | 业务逻辑处理 |
| 数据库操作 | MyBatis | SQL 执行 |
| 数据库 | MySQL | 数据存储 |
| 缓存 | Redis | 用户会话缓存 |
| 认证 | JWT + BCrypt | 登录验证和密码加密 |
| AI 模型 | DeepSeek | AI 智能助手 |
| Excel | EasyExcel | 数据导出 |

### Q5：前后端是怎么通信的？

**建议回答**：
"前端通过 Axios 发送 HTTP 请求到后端。所有 API 请求都带 `/api` 前缀，开发时 Vite 会把 `/api` 的请求代理转发到后端的 8088 端口。后端 Controller 接收请求后调用 Service 处理业务逻辑，最后通过 Mapper 操作数据库。返回的数据统一用 ResultData 格式包装，包含 code（状态码）、msg（提示信息）、data（数据）。"

### Q6：什么是前后端分离？

**建议回答**：
"前后端分离是指前端和后端是两个独立的项目，分别运行在不同的端口上（前端 8080，后端 8088）。前端只负责界面展示和用户交互，后端只负责数据处理和业务逻辑。它们之间通过 HTTP API 通信。这样做的好处是前后端可以独立开发、独立部署，一个后端可以同时服务 Web 端、微信小程序等多个前端。"

## 三、后端技术类问题

### Q7：JWT 登录验证是怎么实现的？

**建议回答**：
"用户登录时，后端验证用户名和密码。验证通过后，用 JJWT 库生成一个令牌（token），令牌里存了 userId 和 username，设置了 2 小时过期时间，用密钥签名。前端拿到令牌后存在 localStorage。之后的每个请求都会在请求头里带上 `Authorization: Bearer <token>`。后端的 JwtAuthenticationFilter 拦截器会解析令牌、验证有效性，从中提取 userId 供后续使用。"

### Q8：密码是怎么存储的？

**建议回答**：
"密码用 BCrypt 加密后存储。BCrypt 是单向加密算法，无法从密文反推出明文。登录验证时，是把用户输入的密码再次 BCrypt 加密，然后和数据库中的密文比对，一致就说明密码正确。这样即使数据库泄露，攻击者也无法获用户的真实密码。"

### Q9：什么是三层架构？

**建议回答**：
"后端代码分为三层：
- **Controller 层**：接收 HTTP 请求，调用 Service，返回响应。代码最薄，只做参数接收和结果返回。
- **Service 层**：真正的业务逻辑在这里。比如入库操作要同时操作多张表、校验数据、生成单号等。
- **Mapper 层**：负责执行 SQL 语句，读写数据库。

分层的目的是让代码职责清晰、好维护。Controller 不需要知道 SQL 怎么写，Mapper 不需要知道业务逻辑。"

### Q10：@Transactional 是做什么的？

**建议回答**：
"`@Transactional` 是 Spring 的事务注解。它保证一个方法里的多个数据库操作'要么全部成功，要么全部失败'。比如入库操作涉及 9 张表的更新，如果其中一步失败，前面已经执行的操作会自动回滚，防止数据不一致。"

### Q11：什么动态 SQL？

**建议回答**：
"MyBatis 支持在 XML 中写动态 SQL，用 `<if>`、`<foreach>` 等标签根据条件拼接 SQL 语句。比如操作日志查询，用户可能传了日期范围也可能没传，动态 SQL 就根据参数是否为 null 来决定要不要加 WHERE 条件。这样不用为每种查询组合写一个单独的 SQL。"

## 四、前端技术类问题

### Q12：Vue 的路由是怎么配置的？

**建议回答**：
"我使用 Vue Router 的 Hash 模式。路由分两类：不需要登录的（/login、/regist）和需要登录的（/home、/manage 等）。需要登录的页面嵌套在 MainLayout 布局中。路由守卫 `beforeEach` 在每次页面跳转前检查是否有 token，没有就重定向到登录页。另外，管理页面还检查用户职位是否是主管。"

### Q13：页面之间怎么传递数据？

**建议回答**：
"主要有三种方式：
1. **localStorage**：存储 token、用户信息等需要持久化的数据，刷新不丢失；
2. **路由参数**：比如从主页跳转 AI 页面时，把问题通过 `$router.push({ query: { q: '...' } })` 传递；
3. **组件内部的 data()**：每个页面管理自己的数据，不与其他页面共享。"

### Q14：Element Plus 是什么？用了哪些组件？

**建议回答**：
"Element Plus 是 Vue 3 的 UI 组件库，提供了现成的按钮、表格、图标、消息提示等组件。我主要用了：
- `<el-icon>`：侧边栏和按钮的图标
- `ElMessage`：操作成功/失败的提示消息
- 表格和弹窗是直接用原生 HTML 写的，配合 Element Plus 的图标"

### Q15：Axios 拦截器做了什么？

**建议回答**：
"我有两个拦截器：
- **请求拦截器**：每次发请求前，自动从 localStorage 取出 token，加到请求头的 Authorization 字段。这样不用每个请求手动加。
- **响应拦截器**：收到响应后，自动提取 `response.data`。普通情况直接返回解析好的数据，Excel 导出时（responseType=blob）保留原始二进制响应。"

## 五、数据库与业务类问题

### Q16：数据库一共有多少张表？核心表有哪些？

**建议回答**：
"一共 22 张表。核心表包括：
- **基础数据**：user（用户）、goods（货物）、warehouse（仓库）、zone（库区）、location（库位）
- **业务数据**：batch（批次）、inventory（库存）
- **单据**：inbound_order_head/detail（入库单）、outbound_order_head/detail（出库单）、adjustment_order_head/detail（调整单）、inventory_check_head/detail（盘点单）、quality_check_head/detail（质检单）、defective_handling_head/detail（次品处理单）
- **辅助**：work_task（工作任务）、operation_log（操作日志）、temperature_humidity_record（温湿度记录）"

### Q17：仓库、库区、库位是什么关系？

**建议回答**：
"这是三级层级关系：一个仓库包含多个库区，一个库区包含多个库位。

- **仓库**：比如'1号冷库'
- **库区**：仓库里的分区，比如'A区'、'B区'
- **库位**：具体的货架格位，比如'A-01-01'

创建仓库时，系统会根据仓库大小自动生成对应数量的库区和库位。大仓库 16 区 128 位，中仓库 8 区 64 位，小仓库 4 区 32 位。"

### Q18：入库操作涉及哪些表的更新？

**建议回答**：
"一次完整入库涉及约 9 张表：
1. `inbound_order_head`：插入入库单头
2. `inbound_order_detail`：插入入库单明细
3. `work_task`：创建工作任务
4. `operation_log`：记录操作日志
5. `inventory`：创建新的库存记录
6. `location`：标记库位已占用
7. `zone`：更新库区可用库位数
8. `warehouse`：更新仓库可用库位数
9. `goods`：更新货物总数量
10. `batch`：更新批次状态

如果任何一步失败，`@Transactional` 事务会回滚所有操作。"

## 六、AI 助手类问题

### Q19：AI 助手是怎么实现的？

**建议回答**：
"AI 助手接入了 DeepSeek 的大语言模型。流程是：

1. 用户输入问题，前端把问题+历史对话发给后端
2. 后端先分析问题关键词（如'库存'、'货物'、'仓库'）
3. 根据关键词查询数据库，获取真实数据
4. 把系统提示词 + 真实数据上下文 + 历史对话 + 用户问题组装成消息
5. 通过 OkHttp 发送 HTTP 请求到 DeepSeek API
6. 解析返回结果中的 AI 回复，返回给前端显示"

### Q20：AI 怎么保证不编造数据？

**建议回答**：
"通过**两层控制**：

1. **系统提示词**中明确要求 AI '严禁编造数据'，如果不知道就告诉用户去哪个页面查看；
2. **真实数据注入**：在调用 AI 之前，后端先查数据库获取真实数据，附在提示词中。这样 AI 看到真实数据后可以基于真实数据回答。比如用户问'冷冻鸡排还有多少？'，后端先查到'200件'，然后在提示词中告诉 AI 这个真实数据。"

## 七、其他常见问题

### Q21：怎么处理跨域问题？

**建议回答**：
"跨域是因为前端（localhost:8080）和后端（localhost:8088）端口不同，浏览器同源策略阻止了请求。我用了两种方式：

1. **开发时**：Vite 的代理功能，把 `/api` 请求转发到后端，浏览器认为是同源请求；
2. **后端**：CorsConfig 配置允许所有来源的跨域请求（生产环境应该限制具体域名）。"

### Q22：项目的配置文件在哪里？配置了什么？

**建议回答**：
"后端配置在 `application.yaml`，主要配置了：

- 服务器端口 8088
- MySQL 数据库连接（地址、用户名、密码）
- Redis 连接
- DeepSeek API 密钥和模型参数
- MyBatis 的 mapper 路径和驼峰命名映射
- 文件上传的路径和大小限制"

### Q23：如果让你改进项目，你会怎么做？

**建议回答**：
"我会从以下几个方面改进：
1. **安全性**：登录改用 POST 方法（目前是 GET，密码在 URL 中不够安全）；**CORS 限制具体域名**；
2. **状态管理**：引入 Pinia 替代 localStorage 管理全局状态，更规范；
3. **权限控制**：**后端也加上权限校验，目前只有前端做了职位判断**；
4. **AI 功能**：使用 SSE（Server-Sent Events）实现流式输出，让 AI 回复逐字显示，体验更好；
5. **代码复用**：把各页面中重复的对话框组件抽取为独立组件。"

### Q24：你在项目中有没有遇到 bug？怎么解决的？

**建议回答**：
"遇到过几个典型的 bug：
1. **JWT 过期问题**：用户操作一段时间后突然提示'令牌失效'，原因是 JWT 2 小时过期后没有刷新机制。解决方法是不刷新页面时重新登录，但这治标不治本，**更好的方案是实现 token 自动刷新**。
2. **跨域问题**：前后端联调时请求被浏览器拦截，排查后发现是 CORS 配置的 Filter 优先级不够高，设置 `Ordered.HIGHEST_PRECEDENCE` 确保 CORS Filter 最先执行就解决了。"
