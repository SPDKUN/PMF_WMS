# 主题切换详解：亮色/暗色模式

> 本项目支持一键切换亮色/暗色主题，使用 CSS 自定义属性 + `data-theme` 属性方案实现，无需引入第三方组件库。

---

## 一、效果演示

- **亮色模式**：白底黑字，绿色侧边栏渐变，适合白天使用
- **暗色模式**：深色背景，浅色文字，侧边栏更深，适合夜间或护眼

切换按钮在顶部栏右侧（太阳/月亮图标），点击即切，刷新页面后保持选择。

## 二、核心原理：CSS 自定义属性

### 什么是 CSS 自定义属性？

CSS 自定义属性（也叫 CSS 变量）允许我们把颜色、尺寸等值定义为变量，然后在任何地方引用。关键是：**变量值可以动态改变**。

```css
:root {
  --card: hsl(0, 0%, 100%);  /* 亮色：白色 */
}

[data-theme="dark"] {
  --card: hsl(160, 5%, 15%);  /* 暗色：深灰 */
}

.card {
  background: var(--card);  /* 自动跟随主题变化 */
}
```

当 `<html>` 标签上有 `data-theme="dark"` 属性时，浏览器自动使用暗色的变量值；没有这个属性时使用默认的亮色值。

### 为什么用 HSL 而不是 HEX？

```css
/* HEX 写法——无法做数学运算 */
--primary: #10b981;

/* HSL 写法——可以调整亮度、饱和度 */
--primary-h: 160;     /* 色相 */
--primary-s: 84%;     /* 饱和度 */
--primary-l: 40%;     /* 亮度 */
--primary: hsl(var(--primary-h), var(--primary-s), var(--primary-l));
```

暗色模式下只需改变 `--primary-l`（亮度），其他保持：

```css
[data-theme="dark"] {
  --primary-l: 50%;  /* 暗色下提高亮度，保持可读性 */
}
```

## 三、代码架构

### 文件结构

```
front/src/
├── style.css              ← 定义 :root 变量 + [data-theme="dark"] 覆盖
├── composables/
│   └── useTheme.js        ← 主题逻辑（读写 localStorage、切换 data-theme）
├── layouts/
│   └── MainLayout.vue     ← 顶部栏中的太阳/月亮切换按钮
└── index.html             ← 预加载脚本，防止初次加载闪烁
```

### 1. CSS 变量定义（style.css）

`style.css` 中 `:root` 块定义了所有设计令牌，按类别组织：

| 类别 | 示例变量 | 亮色值 | 暗色值 |
|------|---------|--------|--------|
| 背景 | `--card` | `hsl(0,0%,100%)` 白色 | `hsl(160,5%,15%)` 深灰 |
| 文字 | `--foreground` | `hsl(0,0%,20%)` 深色 | `hsl(0,0%,93%)` 浅色 |
| 边框 | `--border` | `hsl(160,10%,90%)` 浅灰 | `hsl(160,5%,28%)` 深灰 |
| 阴影 | `--shadow-sm` | 半透明黑色 | 更深的黑色 |
| 主色 | `--primary` | 绿色 | 更亮的绿色 |
| 状态色 | `--success` | 深绿 | 浅绿 |

`[data-theme="dark"]` 块覆盖了所有变量的值，形成了完整的暗色主题。

### 2. 主题组合式函数（useTheme.js）

```javascript
import { ref, watchEffect } from 'vue'

// 从 localStorage 读取上次保存的主题
const saved = localStorage.getItem('app-theme')
const theme = ref(saved === 'dark' ? 'dark' : 'light')

// 当 theme 变化时，自动更新 <html> 的 data-theme 属性 + 写入 localStorage
watchEffect(() => {
  document.documentElement.setAttribute('data-theme', theme.value)
  localStorage.setItem('app-theme', theme.value)
})

export function useTheme() {
  const toggleTheme = () => {
    theme.value = theme.value === 'dark' ? 'light' : 'dark'
  }
  const isDark = () => theme.value === 'dark'
  return { theme, toggleTheme, isDark }
}
```

**关键点**：
- `ref` 创建响应式变量，Vue 会自动追踪它的变化
- `watchEffect` 会在 `theme` 变化时自动执行，不需要手动调用
- `localStorage` 持久化选择，刷新页面不会丢失
- 导出的是**组合式函数**（Composable），Vue 3 的标准代码复用方式

### 3. 切换按钮（MainLayout.vue）

```html
<!-- 模板中：根据当前主题显示太阳或月亮图标 -->
<div class="theme-toggle" @click="toggleTheme"
     :title="theme === 'dark' ? '切换亮色模式' : '切换暗色模式'">
  <el-icon v-if="theme === 'dark'"><Sunny /></el-icon>
  <el-icon v-else><Moon /></el-icon>
</div>
```

```javascript
// 在 Options API 中通过 setup() 使用组合式函数
setup() {
  const { theme, toggleTheme } = useTheme()
  return { theme, toggleTheme }
},
```

`setup()` 是 Vue 3 Options API 中的一个扩展钩子，可以在不迁移到 Composition API 的情况下使用组合式函数。

### 4. 防止加载闪烁（index.html）

Vue 应用需要一定时间才能启动。如果用户之前选择了暗色模式，在 Vue 启动前会看到短暂的白色背景——这就是"闪烁"。

**解决方案**：在 `index.html` 的 `<head>` 中放一段纯 JS 脚本，在 Vue 启动前就读 `localStorage` 并设置 `data-theme` 属性：

```html
<script>
  (function() {
    var theme = localStorage.getItem('app-theme');
    if (theme === 'dark') {
      document.documentElement.setAttribute('data-theme', 'dark');
    }
  })();
</script>
```

这段代码**同步执行**，在页面渲染前就完成了主题设置，用户看不到闪烁。

同时给加载页（`#app-loading`）添加暗色样式：

```css
/* 默认亮色加载页 */
#app-loading { background-color: #f4f7f9; }
#app-loading .title { color: rgb(0 0 0 / 85%); }

/* 暗色模式下的加载页 */
html[data-theme="dark"] #app-loading { background-color: #1a1e1d; }
html[data-theme="dark"] #app-loading .title { color: rgb(255 255 255 / 85%); }
```

## 四、页面组件如何适配暗色模式

### 策略：全局替换硬编码颜色为 CSS 变量

原本页面中有大量硬编码颜色（如 `#303133`、`#606266`、`#f0f0f0`），这些颜色在暗色背景下无法阅读。解决方法是将它们替换为对应的 CSS 变量：

| 硬编码值 | 替换为 | 用途 |
|---------|--------|------|
| `#303133` | `var(--foreground)` | 主要文字 |
| `#606266` | `var(--foreground-regular)` | 次要文字 |
| `#909399` | `var(--foreground-muted)` | 辅助文字 |
| `#c0c4cc` | `var(--foreground-placeholder)` | 占位/禁用文字 |
| `#dcdfe6` | `var(--border)` | 边框色 |
| `#f0f0f0` | `var(--bg-secondary)` | 次要背景 |
| `#fff` | `var(--card)` 或 `var(--text-on-primary)` | 卡片背景 / 按钮文字 |
| `#f0f9eb` | `var(--success-bg)` | 成功状态背景 |
| `#fef0f0` | `var(--danger-bg)` | 错误状态背景 |
| `#f56c6c` | `var(--danger)` | 错误/危险文字 |
| `#67c23a` | `var(--success)` | 成功文字 |

### 保留不动的颜色

图表装饰色（如 `#5ab1ef`、`#10b981`、`#9254de` 等）在两主题下都能接受，不需要替换。这些颜色通常用于 ECharts 图表、任务类型标签、图标背景等装饰性元素。

## 五、你可能被问到的问题

**Q: 为什么选择 CSS 变量方案而不是两套 CSS 文件？**

A: CSS 变量方案只需维护一套代码。添加新组件时自动支持两主题，不需要写两遍样式。运行时切换零延迟，不需要重新加载 CSS 文件。

**Q: `data-theme` 属性为什么放在 `<html>` 上而不是 `<body>`？**

A: 因为 `index.html` 的加载页在 `<body>` 渲染前就需要知道主题，放在 `<html>` 上可以最早生效，避免闪烁。

**Q: `watchEffect` 和 `watch` 有什么区别？**

A: `watchEffect` 会自动追踪回调中用到的所有响应式变量，不需要手动列出依赖。`watch` 需要显式指定要监听的变量。这里用 `watchEffect` 更简洁。

**Q: 为什么不使用 `prefers-color-scheme` 媒体查询？**

A: `prefers-color-scheme` 是浏览器/系统的设置，本项目用 localStorage 手动切换给了用户更多控制权。实际产品中可以将两者结合：默认跟随系统，用户可以覆盖。

**Q: 主题切换会影响页面上的图表（ECharts）吗？**

A: ECharts 图表的颜色是在 JS 中配置的，不受 CSS 变量影响。如果要做图表暗色适配，需要在 JS 中监听主题变化并调用 `chart.setOption()` 更新颜色方案。本项目暂未实现这一点，因为图表中使用的颜色在暗色背景下也可接受。
