# 技术栈介绍：从零理解本项目使用的技术

> 本章面向完全不了解这些技术的小白，用最简单的话解释每一项技术是干什么的。

---

## 一、后端技术

### 1. Java 17 —— 编程语言

Java 是后端使用的编程语言。它的特点是：
- **强类型**：每个变量必须声明类型（如 `int age = 18;`，`String name = "张三";`）
- **面向对象**：一切皆为"对象"，代码围绕"类"（class）来组织
- **跨平台**：一次编写，到处运行（依赖 JVM 虚拟机）

**在本项目中的作用**：所有后端业务逻辑都是用 Java 写的。

---

### 2. Spring Boot 3 —— 后端框架（核心！）

Spring Boot 是 Java 世界最流行的后端框架。你可以把它理解为**一个已经搭好的"骨架"**，帮我们处理了：
- 启动服务器（监听 8088 端口）
- 接收 HTTP 请求
- 管理对象（IoC 容器，自动 new 对象）
- 连接数据库
- 返回 JSON 数据

**在本项目中的作用**：它是整个后端的"大管家"，所有功能都是挂载在 Spring Boot 上的。

**怎么识别**：`@SpringBootApplication` 注解标注的类就是项目的入口。

---

### 3. MyBatis —— 数据库操作框架

MyBatis 是一个简化数据库操作的框架。它的作用是：
- 把 Java 对象和数据库表对应起来
- 让我们用简单的方法调用就能执行 SQL 语句
- 不用手写 JDBC 那套繁琐的代码

工作原理：
```
Java方法调用  →  Mapper接口  →  XML中的SQL  →  MySQL数据库
```

例如，调用 `userMapper.findById(1)` 就会执行 `SELECT * FROM user WHERE user_id = 1`。

**在本项目中的作用**：所有的增删改查都通过 MyBatis 完成。

**怎么识别**：
- `mapper/` 目录下的接口（如 `UserMapper.java`）
- `mapper/*.xml` 文件（里面写着真正的 SQL 语句）

---

### 4. MySQL —— 关系型数据库

MySQL 是一个存储数据的软件。数据以"表格"的形式存储，就像 Excel 表格一样。

例如 `user` 表：
| user_id | username | real_name | position |
|---------|----------|-----------|----------|
| 1 | admin | 张三 | 主管 |
| 2 | staff01 | 李四 | 仓管员 |

**在本项目中的作用**：存储所有业务数据，包括用户、货物、仓库、出入库记录等。

---

### 5. JWT (JSON Web Token) —— 登录验证

JWT 是一种"身份令牌"机制。

**为什么需要它？** **HTTP 请求是"无状态"的**，也就是说服务器不知道两次请求来自同一个用户。所以需要一个令牌来证明"我是谁"。

**工作流程**：
```
1. 用户登录 → 服务器验证用户名密码 → 服务器返回一个JWT令牌（一串加密字符串）
2. 用户之后的每次请求，都在请求头带上这个令牌
3. 服务器读取令牌，就知道这个请求是哪个用户发的
```

**令牌长什么样**：
```
eyJhbGciOiJIUzM4NCJ9.eyJ1c2VySWQiOjEsInVzZXJuYW1lIjoiYWRtaW4i...（很长一串）
```

这不是乱码，而是经过签名加密的 JSON 数据。解密后**里面存着 `userId`（用户ID）和 `username`（用户名）**。

**在本项目中的作用**：

- 登录时生成令牌返回给前端
- 前端把令牌存在 `localStorage` 中
- 每次发请求时在请求头带上 `Authorization: Bearer <token>`
- 后端的 `JwtAuthenticationFilter` 过滤器会验证令牌是否有效

---

### 6. BCrypt —— 密码加密

BCrypt 是一种**单向加密算法**。用户的密码绝对不会明文存储在数据库中。

比如用户密码是 `123456`，数据库里存的是一串乱码：`$2a$10$N9qo8uLOickgx2ZMRZoMyeIjZAgcfl7p92ldGxad68LJZdL17lhWy`

**验证密码的方式不是"解密"，而是"再次加密后比对"**。

**在本项目中的作用**：注册和修改密码时用 BCrypt 加密存储；登录时用 BCrypt 比对密码是否匹配。

---

### 7. Redis —— 缓存数据库

Redis 是一个**内存级别的键值存储**。它的特点是**极快**，因为数据存在内存中。

**在本项目中的作用**：用户登录成功后，把用户信息缓存到 Redis，下次请求时直接从 Redis 读取，不用每次都去 MySQL 查。

---

### 8. EasyExcel —— Excel 导入导出

阿里巴巴开源的库，用于读写 Excel 文件。

**在本项目中的作用**：明细查询页面可以导出各种数据（用户列表、货物列表、操作日志等）为 Excel 文件。

---

### 9. OkHttp —— HTTP 客户端

一个发送 HTTP 请求的 Java 库。

**在本项目中的作用**：后端调用 DeepSeek AI 的 API 时，用 OkHttp 发送请求。相当于后端的"axios"。

---

### 10. SpringDoc OpenAPI (Swagger) —— API 文档

自动根据代码生成 API 接口文档，可以在浏览器中查看和测试所有接口。

---

## 二、前端技术

### 1. Vue 3 —— 前端框架（核心！）

Vue 是一个构建用户界面的 JavaScript 框架。它的核心思想是**数据驱动视图**。

**通俗理解**：传统方式你需要手动操作 DOM（`document.getElementById(...)`）来更新页面。而 Vue 让你只需要修改数据，页面会自动更新。

**核心概念**：
- **template（模板）**：写 HTML 的地方
- **script（脚本）**：写 JS 逻辑的地方（data 定义数据，methods 定义方法）
- **style（样式）**：写 CSS 的地方
- **v-model**：双向绑定，**输入框的值会自动同步到 JS 变量**
- **v-if / v-for**：条件渲染和列表渲染
- **{{ }}（插值表达式）**：在 HTML 中显示 JS 变量的值

**例子**：
```html
<template>
  <div>
    <input v-model="username" />  <!-- 输入的内容自动存到 username 变量 -->
    <p>你好，{{ username }}</p>   <!-- 自动显示 username 的值 -->
  </div>
</template>
```

**在本项目中的作用**：整个前端的页面都是用 Vue 3 写的。

---

### 2. Element Plus —— UI 组件库

Element Plus 是一套现成的 UI 组件库，提供了按钮、表格、弹窗、表单等常用组件。

**通俗理解**：不用自己写 CSS 画按钮、画表格，直接用 `<el-button>按钮</el-button>`、`<el-table>` 就能得到一个漂亮的组件。

**在本项目中的作用**：侧边栏的图标、表格显示、弹窗、消息提示等都用了 Element Plus。

---

### 3. Vue Router —— 路由管理

管理页面之间的跳转。用户点击侧边栏的不同菜单，URL 改变，显示不同的页面。

**在本项目中的作用**：定义了登录页、主页、管理页等 8 个路由，以及路由守卫（没有登录就跳回登录页）。

---

### 4. Axios —— 发送 HTTP 请求

前端用来跟后端通信的库。封装了浏览器原生的 `fetch` API，用起来更方便。

**在本项目中的作用**：所有前端页面通过 axios 向后端发送请求（如获取用户列表、创建入库任务等）。

---

### 5. ECharts —— 数据可视化图表库

百度开源的图表库，可以画折线图、柱状图、饼图等。

**在本项目中的作用**：数据看板页面的出入库趋势图、货物数量柱状图、温湿度图表。

---

### 6. Vite —— 构建工具

前端的"打包器"。开发时启动一个本地服务器（端口 8080），修改代码后页面自动刷新。发布时把代码打包成浏览器可直接运行的 HTML/JS/CSS 文件。

---

## 三、总结：一张图看清技术栈

```
                    前端 (Vue 3 + Element Plus)
                    运行在浏览器中
                    ↓
        用户操作 → Axios发送HTTP请求
                    ↓
         ┌──────────────────────┐
         │  Nginx / Vite Proxy  │  ← 转发 /api 请求到后端
         └──────────────────────┘
                    ↓
              后端 (Spring Boot)
              运行在 8088 端口
                    ↓
         ┌──────────────────────┐
         │   JWT Filter 验证    │  ← 检查请求有没有合法令牌
         └──────────────────────┘
                    ↓
         Controller → Service → Mapper
                    ↓
         ┌─────────┬──────────┐
         │  MySQL  │  Redis   │  ← 数据存储
         └─────────┴──────────┘
                    ↓
         需要AI时 → OkHttp → DeepSeek API
```
