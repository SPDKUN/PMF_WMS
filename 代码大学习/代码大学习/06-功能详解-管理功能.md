# 功能详解三：管理功能（人员、仓库、货物）

> 这是系统的"后台管理"模块，只有主管（position='主管'）能访问。

## 一、页面概述

### 页面位置
- 前端：`front/src/pages/manage/index.vue`

页面顶部有三个选项卡，对应三个管理对象：

```
┌─────────────────────────────────────────────┐
```

三个选项卡分别管理三张数据库表：`user`、`warehouse`、`goods`。

## 二、人员管理

### 涉及的后端接口（增POST删DEL改PUT查GET）

| 操作 | HTTP方法 | 接口路径 |
|------|---------|---------|
| 获取人员列表 | GET | `/user/personnel` |
| 新增人员 | POST | `/user` |
| 修改人员信息 | PUT | `/user` |
| 删除人员 | DELETE | `/user/{id}` |

### 新增人员的前端代码（关键部分）

```javascript
// 弹出新增对话框，用户填写表单后点击确认
async confirmCreateUser() {
  // 向后端发送 POST 请求，body 是表单数据
  await request.post('/user', {
    username: this.form.username,
    password: this.form.password,
    real_name: this.form.realName,
    phone: this.form.phone,
    department: this.form.department,
    position: this.form.position
  })
  ElMessage.success('新增人员成功')
  this.loadPersonnel()  // 刷新列表
}
```

### 后端新增人员流程

```java
// UserController.java
@PostMapping
public ResultData addUser(@RequestBody User user) {
    userService.addUser(user);
    return ResultData.success();
}

// UserServiceImpl.java
public void addUser(User user) {
    // 用 BCrypt 加密密码
    user.setPassword(bCryptPasswordEncoder.encode(user.getPassword()));
    // 存入数据库
    userMapper.insert(user);
}
```

**关键点**：后端接收到的密码是明文（如 "123456"），但在存入数据库之前用 BCrypt 加密。数据库中存储的密码是不可读的密文。

### 修改和删除

修改和删除都类似：前端发送请求 → 后端更新/删除数据库记录 → 刷新列表。

删除用的是 HTTP DELETE 方法，路径是 `/user/{id}`（如 `/user/5` 表示删除 ID 为 5 的用户）。

## 三、仓库管理

### 涉及的后端接口

| 操作 | HTTP方法 | 接口路径 |
|------|---------|---------|
| 获取仓库列表 | GET | `/warehouse/list` |
| 新增仓库 | POST | `/warehouse` |
| 修改仓库 | PUT | `/warehouse` |
| 删除仓库 | DELETE | `/warehouse/{id}` |

### 创建仓库时的自动建区建位

这是仓库管理中最**重要**的业务逻辑。创建仓库时，系统会自动创建库区和库位：

```java
// WarehouseServiceImpl
// 根据仓库大小自动生成库区和库位
switch (warehouseSize) {
    case "大":  // 16个库区，128个库位
        createZones(warehouseId, 16);
        createLocations(zoneIds, 128);
        break;
    case "中":  // 8个库区，64个库位
        createZones(warehouseId, 8);
        createLocations(zoneIds, 64);
        break;
    case "小":  // 4个库区，32个库位
        createZones(warehouseId, 4);
        createLocations(zoneIds, 32);
        break;
}
```

**为什么要这样做？** 仓库管理的层级关系是：

```
仓库（Warehouse）
 └── 库区（Zone）      ← 仓库里的分区，如"A区"、"B区"
      └── 库位（Location） ← 具体的货架位置，如"A-01-01"
```

创建仓库时如果不自动建区建位，管理员还得手动一个一个添加，非常麻烦。

## 五、三个选项卡如何切换？

```html
<div class="tab-row">
  <button :class="{ active: activeTab === 'personnel' }"
          @click="activeTab = 'personnel'">人员管理</button>
  <button :class="{ active: activeTab === 'warehouse' }"
          @click="activeTab = 'warehouse'">仓库管理</button>
  <button :class="{ active: activeTab === 'goods' }"
          @click="activeTab = 'goods'">货物管理</button>
</div>

<!-- 根据 activeTab 的值显示对应的内容 -->
<div v-if="activeTab === 'personnel'">人员表格...</div>
<div v-if="activeTab === 'warehouse'">仓库表格...</div>
<div v-if="activeTab === 'goods'">货物表格...</div>
```

使用一个 `activeTab` 变量配合 `v-if` 来控制显示哪个选项卡的内容。这是一种非常常见的前端模式。

## 六、你可能被问到的问题

**Q: 创建仓库时，库区和库位是怎么生成的？**

A: 根据仓库的大小等级（大/中/小）自动生成。大型仓库创建 16 个库区和 128 个库位，中型 8 区 64 位，小型 4 区 32 位。库区命名如"A区"、"B区"，库位命名如"A-01-01"。

**Q: 为什么密码不能明文存储？**

A: 如果数据库被泄露，明文密码会直接暴露用户的真实密码。使用 BCrypt 加密后，即便数据库被获取，攻击者也无法还原出明文密码。BCrypt 是单向加密，不可逆。

**Q: 修改用户信息时如果不输入密码怎么办？**

A: 在后端 UserServiceImpl 的 update 方法中，如果传过来的密码为空，就保留原密码不变；如果传了新密码，就用 BCrypt 加密后替换。
