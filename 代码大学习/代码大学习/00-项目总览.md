# 项目总览：预制菜 WMS 智能仓储管理系统

## 一、这个项目是什么？

这是一个 **WMS（Warehouse Management System，仓库管理系统）**，专门为**预制菜行业**设计。

通俗理解：一家预制菜工厂有很多仓库，仓库里有很多货物（比如冷冻鸡排、冷藏蔬菜等）。工人每天要做这些事情：
- 货物生产出来后**入库**（放进仓库）
- 接到订单后**出库**（从仓库取出来）
- 定期**盘点**（数一数实际库存和系统记录是否一致）
- **质检**（检查货物有没有变质）
- 处理**次品**（坏掉的货物）
- 记录仓库的**温度和湿度**

这个系统就是用软件来代替纸笔，把这些事情全部管理起来。

## 二、系统的整体架构

整个系统分为两大部分：

```
┌──────────────────────┐         ┌──────────────────────┐
│    前端 (front)       │  HTTP   │    后端 (backend)     │
│  用户看到的网页界面    │ <-----> │  处理业务逻辑的服务器   │
│                      │  JSON   │                      │
│  Vue 3 + Element Plus │         │  Spring Boot + MyBatis │
└──────────────────────┘         └──────────┬───────────┘
                                            │
                                            │ SQL
                                            ▼
                                   ┌──────────────────┐
                                   │   MySQL 数据库    │
                                   │   存储所有数据     │
                                   └──────────────────┘
```

**前端**：用户用浏览器打开的网页，负责显示界面、收集用户输入。

**后端**：运行在服务器上的 Java 程序，负责处理业务逻辑、操作数据库。

**数据库**：MySQL，存储用户信息、货物信息、仓库信息、出入库记录等所有数据。

## 三、系统有哪些功能？

系统共有 **8 个主要页面**：（前端pages文件夹）

| 页面 | 路由 | 能做什么 |
|------|------|---------|
| 登录页 | `/login` | 用户输入用户名密码登录 |
| 注册页 | `/regist` | 新用户注册账号 |
| 主页 | `/home` | 显示统计数据概览、待办提醒、快捷AI入口 |
| 数据看板 | `/dashboard` | 用图表展示出入库趋势、货物数量、温湿度等 |
| 我的管理 | `/manage` | 管理人员、仓库、货物的增删改查（仅主管可用） |
| 明细查询 | `/query` | 查看各种数据列表、导出Excel |
| 工作任务 | `/tasks` | 查看待办任务、创建出入库/质检/盘点等操作 |
| AI助手 | `/ai` | 向AI提问仓储相关问题 |

## 四、项目中的角色

系统中有不同的**职位**（position）：

- **主管**：权限最高，可以进入"我的管理"页面进行人员、仓库、货物的管理
- **仓库员工**：负责日常的出入库操作
- **质检员**：负责质量检查
- **盘点员**：负责库存盘点

## 六、怎样运行这个项目？

**后端**：
1. 确保电脑安装了 Java 17 和 MySQL
2. 在 MySQL 中创建 `pmf` 数据库，运行建表SQL
3. 启动 Redis（用于缓存）
4. 用 Maven 启动 Spring Boot 项目，默认端口 8088

**前端**：
1. 确保电脑安装了 Node.js
2. 在 `front` 目录下运行 `npm install` 安装依赖
3. 运行 `npm run dev` 启动开发服务器
4. 浏览器访问 `http://localhost:8080`

> 前端开发服务器会自动把 `/api` 开头的请求转发到后端 8088 端口，这就是前后端分离项目的典型做法。
