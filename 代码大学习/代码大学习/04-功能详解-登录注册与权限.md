# 功能详解一：登录、注册与权限控制

> 这是用户最先接触的功能。看似简单，但涉及前后端协作、密码加密、令牌管理等多个知识点。

---

## 一、登录功能

### 用户操作流程

```
用户打开网页 → 看到登录页 → 输入用户名密码 → 点击登录
  → 成功：跳转到主页
  → 失败：显示"用户名或密码错误"
```

### 前端代码解析（login/index.vue）

#### 1. 数据定义

```javascript
data() {
  return {
    username: '',    // 输入的用户名
    password: '',    // 输入的密码
    errorMsg: '',    // 错误提示信息
    loading: false   // 是否正在登录中（控制按钮禁用）
  }
}
```

#### 2. 表单的双向绑定

```html
<input v-model="username" type="text" placeholder="请输入用户名" />
<input v-model="password" type="password" placeholder="请输入密码" />
```

**`v-model` 让输入框和 JS 变量实时同步。用户在输入框里打字，`username` 变量就会自动更新。**

#### 3. 登录按钮的状态切换

```html
<button type="submit" :disabled="loading">
  <span v-if="!loading">登 录</span>
  <span v-else>登录中...</span>
</button>
```

- `:disabled="loading"` —— 当 `loading=true` 时按钮禁用，防止重复点击
- **`v-if="!loading"` —— 正常时显示"登 录"**
- **`v-else` —— 加载时显示"登录中..."**

#### 4. 核心登录逻辑

```javascript
async handleLogin() {
  this.errorMsg = ''
  this.loading = true                        // 步骤1：显示加载状态
  try {
    const response = await request.get('/auth/login', {
      username: this.username,               // 步骤2：发送GET请求
      password: this.password
    })

    const token = response.token
    if (token) {
      const userData = response.data
      // 步骤3：检查账号是否被停用（status=0）
      if (userData && userData.status === 0) {
        this.errorMsg = '该账号已被停用，请联系管理员'
        return
      }
      // 步骤4：保存令牌和用户信息到本地
      localStorage.setItem('token', token)
      localStorage.setItem('userInfo', JSON.stringify(userData))

      // 步骤5：生成AI会话ID
      const sessionId = Date.now().toString()
      localStorage.setItem('ai_session_id', sessionId)

      // 步骤6：跳转主页
      this.$router.push({ name: 'Home' })
    }
  } catch (error) {
    this.errorMsg = '网络问题或者用户名密码错误'  // 步骤7：错误处理
  } finally {
    this.loading = false                       // 步骤8：恢复按钮
  }
}
```

**三个关键点**：
1. **为什么要 `async/await`？** 因为网络请求需要时间（几百毫秒），JS 是异步的，用 `await` 等待结果回来再继续执行。
2. **为什么用 `try/catch`？** 网络请求可能失败（服务器挂了、密码错了等），需要捕获错误防止页面崩溃。
3. **为什么用 `localStorage`？** 浏览器刷新后，JS 变量的数据会丢失。`localStorage` 里的数据是持久化的，刷新页面也不会丢。

### 后端代码解析（AuthController.java）

#### Controller 层：接收请求

```java
@RequestMapping("/auth")
@RestController
public class AuthController {

    @GetMapping("/login")
    public ResultData login(
        @RequestParam("username") String username,
        @RequestParam("password") String password
    ) {
        // 调用 Service 验证用户
        User user = authService.login(username, password);

        if (user != null) {
            // 验证成功 → 生成 JWT 令牌
            String token = jwtUtil.generateToken(user.getUser_id(), username);

            // 把令牌放入返回结果
            ResultData resultData = ResultData.success(user);
            resultData.put("token", token);
            return resultData;
        }

        // 验证失败
        return ResultData.fail(ResultCode.USER_AND_PASSWORD_ERROR);
    }
}
```

**注意**：登录用的是 `@GetMapping`（而非常规的 `@PostMapping`）。这意味着参数通过 URL 传递：`/auth/login?username=admin&password=123456`。这种做法在安全性上不够好（密码暴露在 URL 中），但对于这个学习项目是可以接受的。

#### Service 层：真正的登录验证逻辑

AuthServiceImpl 做了这些事：
1. 根据用户名去数据库查用户记录
2. 用 **BCrypt** 比对密码（不是明文比对！）
3. 如果匹配成功：
   - 把用户信息存入 Redis 缓存
   - 返回 User 对象
4. 如果匹配失败：返回 null

## 二、JWT 令牌详解

### 令牌的生成

```java
public String generateToken(Long userId, String username) {
    // 1. 创建签名密钥（密钥来自 JwtConstants，是一段固定的字符串）
    SecretKey secretKey = Keys.hmacShaKeyFor(JwtConstants.JWT_SECRET.getBytes());

    // 2. 构建 Payload（载荷，即令牌中要存储的数据）
    Map<String, Object> claims = new HashMap<>();
    claims.put("userId", userId);       // 用户ID
    claims.put("username", username);   // 用户名

    // 3. 组装并签名
    return Jwts.builder()
        .setClaims(claims)
        .setIssuedAt(new Date())        // 签发时间
        .setExpiration(new Date(        // 过期时间（当前时间 + 2小时）
            System.currentTimeMillis() + JwtConstants.JWT_EXPIRE_TIME * 1000))
        .signWith(secretKey)            // 签名
        .compact();                     // 生成最终的字符串
}
```

### 令牌的验证（每次请求时执行）

JwtAuthenticationFilter 在每次请求时都做这件事：
1. 从请求头提取 `Authorization: Bearer <token>`
2. 去掉 `Bearer ` 前缀，得到纯令牌
3. 用同样的密钥尝试解析令牌
4. 如果解析成功 → 令牌有效，放行
5. 如果解析失败 → 令牌无效或过期 → 返回 `{"code": 1002, "msg": "令牌已过期或者失效"}`

```javascript
await axios.post('/api/user', {

```

后端 UserController 收到后，用 BCrypt 加密密码，存到数据库。

## 四、权限控制：谁能看什么页面

### 前端权限（路由守卫）

```javascript
// 普通保护：需要登录
{ path: 'home', meta: { requiresAuth: true } }

// 职位保护：主管才能访问
{ path: 'manage', meta: { requiresAuth: true, requirePosition: '主管' } }
```

### 前端权限（菜单显示）

```html
<!-- 侧边栏中，"我的管理"菜单仅主管可见 -->
<a v-if="userPosition === '主管'" @click="navigate('Manage')">我的管理</a>
```

### 后端权限

后端的 JWT 过滤器做身份验证（你是谁），但没有做权限验证（你能做什么）。所有已登录用户都能调用任何接口。这是这个项目的简化之处。

## 五、退出登录

```javascript
handleLogout() {
  // 清除AI聊天记录
  localStorage.removeItem('ai_chat_msg_' + username + '_' + sessionId)
  // 清除令牌
  localStorage.removeItem('token')
  // 清除用户信息
  localStorage.removeItem('userInfo')
  // 清除AI会话
  localStorage.removeItem('ai_session_id')
  // 跳转登录页
  this.$router.push({ name: 'Login' })
}
```

退出登录非常简单：**清除浏览器中存储的令牌和用户信息，然后跳回登录页**。后端不需要做什么，因为 JWT 的"退出"就是客户端自己把令牌丢掉。

## 六、你可能被问到的问题

**Q: 用户登录后，后端如何知道下一个请求是谁发的？**

A: 登录时后端返回一个 JWT 令牌。前端把令牌存在 localStorage。之后的每个请求都在请求头中带上 `Authorization: Bearer <token>`。后端的 JwtAuthenticationFilter 拦截器解析令牌，从中提取 userId，就知道是哪个用户了。

**Q: JWT 令牌过期了怎么办？**

A: 本项目设置的是 2 小时过期。过期后用户的操作会失败（返回 "令牌已过期"）。但在本项目中**没有做自动刷新令牌**或友好提示，用户需要重新登录。

**Q: 密码在数据库中是怎么存的？**

A: 用 BCrypt 加密后存储。即便是数据库管理员也看不到用户的明文密码。**BCrypt 是单向加密，无法反推出原文。**验证时是"把输入的密码再次加密，比对加密结果是否一致"。
