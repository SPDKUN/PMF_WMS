# 前端架构详解：Vue 3 项目是如何组织的

> 读完本章，你将理解前端项目的文件结构、页面渲染流程，以及前端如何与后端通信。

## 一、前端项目的入口

### 1.1 index.html → main.js → App.vue

一切从这个链条开始：

```
index.html (浏览器加载的HTML文件)
  └── 加载 main.js (项目入口JS文件)
        └── 创建 Vue 应用 → 挂载路由 → 渲染 App.vue
              └── App.vue 里面只有 <RouterView>
                    └── 根据当前URL显示对应的页面组件
```

### 1.2 main.js 做了什么

```javascript
import { createApp } from 'vue'      // 导入 Vue 框架
import router from './router'        // 导入路由配置
import './style.css'                 // 导入全局样式
import 'element-plus/dist/index.css' // 导入 Element Plus 组件库样式
import App from './App.vue'          // 导入根组件

const app = createApp(App)  // 创建一个 Vue 应用
app.use(router)             // 注册路由
app.mount('#app')           // 挂载到 index.html 中的 <div id="app">
```

位置：`front/src/main.js`

## 二、路由系统（最重要！）

### 2.1 什么是路由？

路由决定了**URL 和页面的对应关系**。

```
用户访问 http://localhost:5173/#/login   →  显示 Login 页面
用户访问 http://localhost:5173/#/home    →  显示 Home 页面
用户访问 http://localhost:5173/#/tasks   →  显示 Tasks 页面
```

注意 `#` 号——这是 **Hash 模式**的路由，`#` 后面的部分不会发送给服务器，只在浏览器端处理。

### 2.2 路由配置详解

```javascript
const routes = [
  // 不需要登录就能访问的页面
  { path: '/login', name: 'Login', component: Login },
  { path: '/regist', name: 'Regist', component: Regist },

  // 需要登录才能访问的页面（嵌套在 MainLayout 中）
  {
    path: '/',
    component: MainLayout,    // 外层布局（侧边栏 + 顶栏）
    redirect: '/login',       // 访问根路径时跳转到登录页
    children: [
      {
        path: 'home',
        name: 'Home',
        component: Home,
        meta: { requiresAuth: true }  // ← 需要登录
      },
      {
        path: 'manage',
        name: 'Manage',
        component: Manage,
        meta: { requiresAuth: true, requirePosition: '主管' }  // ← 主管才能访问
      },
      // ... 其他页面
    ]
  }
]
```

位置：`front/src/router/index.js`

### 2.3 路由守卫：没有登录就拦下来

```javascript
router.beforeEach((to, from, next) => {
  const token = localStorage.getItem('token')  // 检查有没有令牌

  if (to.matched.some(record => record.meta.requiresAuth)) {
    // 这个页面需要登录
    if (!token) {
      next({ name: 'Login' })   // 没令牌 → 跳转登录页
      return
    }

    // 检查是否需要特定职位
    if (to.meta.requirePosition) {
      const user = JSON.parse(localStorage.getItem('userInfo'))
      if (user.position !== to.meta.requirePosition) {
        next({ name: 'Home' })  // 不是主管 → 跳转主页
        return
      }
    }
    next()  // 一切正常，放行
  } else {
    next()  // 不需要登录的页面，直接放行
  }
})
```

**路由守卫的本质**：就像公司门口的保安。每次有人想进（页面跳转），就检查他有没有门禁卡（token）。

## 三、MainLayout：页面骨架

```html
<aside class="sidebar">     <!-- 左侧：侧边栏 -->
  <div class="logo">WMS</div>
  <nav>                     <!-- 导航菜单 -->
    <a>主页</a>
    <a>我的管理</a>          <!-- 仅主管可见 -->
    <a>数据看板</a>
    <a>明细查询</a>
    <a>工作任务</a>
    <a>AI助手</a>
    <a>个人中心</a>
  </nav>
  <div class="logout-btn">  <!-- 退出按钮 -->
</aside>

<main class="main">         <!-- 右侧：主内容区 -->
  <header class="topbar">   <!-- 顶栏：页面标题 + 日期 -->
  </header>
  <router-view></router-view>  <!-- ★ 这里显示子页面 ★ -->
</main>
```

位置：`front/src/layouts/MainLayout.vue`

**路由嵌套的原理**：当用户访问 `/home` 时，Vue Router 会把 `Home` 组件渲染到 `<router-view>` 的位置。而外面的侧边栏和顶栏始终不变。

```
┌────────────┬──────────────────────────┐
│  侧边栏     │  顶栏（标题 + 日期）       │
│            ├──────────────────────────┤
│  主页       │                          │
│  管理       │     <router-view>        │
│  看板       │     这里显示当前页面       │
│  查询       │                          │
│  任务       │                          │
│  AI        │                          │
│  个人       │                          │
│            │                          │
│  [退出]    │                          │
└────────────┴──────────────────────────┘
```

### "我的管理"菜单的条件显示

```javascript
// 读取用户信息，判断是否是主管
const stored = localStorage.getItem('userInfo')
```

## 四、如何与后端通信：request.js

前端不能直接操作数据库，所有数据都要通过 HTTP 请求向**后端**获取。

```javascript
import axios from 'axios'

// 创建一个配置好的 axios 实例
const service = axios.create({
  baseURL: '/api',     // 所有请求自动加上 /api 前缀
  timeout: 10000,      // 10秒超时
})

// ★ 请求拦截器：每次发请求前自动执行
service.interceptors.request.use((config) => {
  const token = localStorage.getItem('token')
  if (token) {
    config.headers.Authorization = `Bearer ${token}`  // 自动带令牌
  }
  return config
})

// ★ 响应拦截器：每次收到响应后自动执行
service.interceptors.response.use((response) => {
  if (response.config.responseType === "blob") {
    return response        // 文件下载时返回原始响应
  }
  return response.data     // 普通请求只返回 data（即后端的 ResultData）
})
```

位置：`front/src/utils/request.js`

### 请求拦截器的作用

不需要在每个页面手动加 token，axios 会自动帮你加：

```javascript
// 不拦截器的话，每个请求都要这样写：
request.get('/user/personnel', {
  headers: { Authorization: 'Bearer ' + localStorage.getItem('token') }
})

// 有了拦截器，直接这样写就行：
request.get('/user/personnel')
```

### 响应拦截器的作用

后端返回的是 `{ code: 200, msg: "操作成功", data: [...] }`，拦截器自动提取 `.data`，所以在页面代码中：

```javascript
const res = await request.get('/user/personnel')
// res 直接就是 { code: 200, msg: "操作成功", data: [...] }
// 不需要 res.data
```

### 封装的请求方法

```javascript
const request = {
  get(url, params) { ... },          // GET 请求
  post(url, data) { ... },          // POST 请求
  put(url, data) { ... },           // PUT 请求
  delete(url, params) { ... },      // DELETE 请求
  upload(url, file, params) { ... },// 文件上传
  exportExcel(url, fileName) { ... }// Excel 导出（下载文件）
}
```

## 五、Vue 页面是如何工作的

以**登录页面**为例，讲解一个 Vue 页面的结构：

```html
<!-- 1. template：写HTML -->
<template>
  <div class="login-page">
    <form @submit.prevent="handleLogin">
      <input v-model="username" placeholder="请输入用户名" />
      <input v-model="password" type="password" placeholder="请输入密码" />
      <button type="submit" :disabled="loading">
        <span v-if="!loading">登 录</span>
        <span v-else>登录中...</span>
      </button>
    </form>
    <p v-if="errorMsg" class="error-msg">{{ errorMsg }}</p>
  </div>
</template>

<!-- 2. script：写逻辑 -->
<script>
export default {
  name: 'LoginView',
  data() {                         // 定义数据（响应式，变了页面自动更新）
    return {
      username: '',                 // 用户名
      password: '',                 // 密码
      errorMsg: '',                 // 错误消息
      loading: false                // 是否正在加载
    }
  },
  methods: {                        // 定义方法
    async handleLogin() {
      this.loading = true           // 显示"登录中..."
      try {
        const response = await request.get('/auth/login', {
          username: this.username,
          password: this.password
        })
        // 登录成功，保存 token，跳转主页
        localStorage.setItem('token', response.token)
        this.$router.push({ name: 'Home' })
      } catch (error) {
        this.errorMsg = '网络问题或者用户名密码错误'
      } finally {
        this.loading = false
      }
    }
  }
}
</script>

<!-- 3. style：写CSS（scoped表示样式只在这个组件生效） -->
<style scoped>
.login-page { ... }
.login-btn { ... }
</style>
```

### Vue 的关键语法速查

| 语法 | 作用 | 例子 |
|------|------|------|
| `{{ 变量 }}` | 在HTML中显示JS变量的值 | `<p>{{ username }}</p>` |
| `v-model="变量"` | 双向绑定（输入框↔变量） | `<input v-model="username" />` |
| `v-if="条件"` | 条件渲染（条件为false时不显示） | `<p v-if="errorMsg">错误</p>` |
| `v-for="item in list"` | 列表渲染（循环） | `<tr v-for="user in userList">` |
| `@click="方法"` | 点击事件 | `<button @click="handleLogin">` |
| `:disabled="变量"` | 动态属性绑定 | `<button :disabled="loading">` |

## 六、前端的状态管理

本项目**没有使用 Vuex/Pinia**，而是用最简单的方案：

### localStorage（浏览器本地存储）

```javascript
// 存
localStorage.setItem('token', token)
localStorage.setItem('userInfo', JSON.stringify(userData))

// 取
const token = localStorage.getItem('token')
const user = JSON.parse(localStorage.getItem('userInfo'))
```

| 存储的键 | 内容 | 用途 |
|---------|------|------|
| `token` | JWT令牌字符串 | 每次请求带上，证明身份 |
| `userInfo` | 用户信息的JSON | 显示用户名、判断权限 |
| `ai_session_id` | 会话ID | AI聊天记录按会话隔离 |
| `sidebarCollapsed` | 布尔值 | 侧边栏是否收缩 |

### 组件内部 data()

每个页面自己管理自己的数据，页面的数据在 `data()` 中定义，不与其他页面共享。这是最简单的状态管理方式，对于这个规模的项目来说完全够用。

## 七、前端如何应对"后端还没启动"

前端开发时，Vite 会自动启动一个代理服务器：

```javascript
// vite.config.js 中（隐式配置）
// 当请求 /api/xxx 时，Vite 自动转发到 http://localhost:8088
```

所以前端代码中写 `request.get('/user/personnel')`，实际请求的是：
- 开发环境：`http://localhost:5173/api/user/personnel` → Vite 转发到 `http://localhost:8088/user/personnel`
- 生产环境：前端和后端部署在同一域名下，直接发送到后端
