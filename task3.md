## ipconfig 分析结果

### 网络适配器状态一览

| 适配器 | 状态 | IPv4 | 能否用？ |
|--------|------|------|----------|
| SV-Connection | ❌ 断开 | - | |
| 以太网 4 | ✅ | 26.189.221.229 | 可能（公网/隧道IP） |
| 以太网 | ❌ 断开 | - | |
| **本地连接\* 1** | ❌ **断开** | - | 热点未开启！ |
| **本地连接\* 2** | ❌ **断开** | - | 热点未开启！ |
| VMnet1 | ✅ | 192.168.11.1 | ❌ VMware虚拟机 |
| VMnet8 | ✅ | 10.0.0.1 | ❌ VMware NAT |
| **WLAN** | ✅ | **10.242.194.76** | ✅ **用这个！** |

### 核心结论

**你的热点根本没开！** 两个热点适配器（本地连接\* 1、本地连接\* 2）全部断开。当前唯一活跃的无线网络是 **WLAN，IP = 10.242.194.76**。

解决方案就一个：**手机和电脑连同一个 WiFi**，然后手机就能访问 `10.242.194.76:8088`。

## 你需要手动做的事情

### 必须做的：

**1. 手机连上电脑同一个 WiFi**
   - 电脑当前连的 WiFi 分配了 `10.242.194.76`
   - 手机也连这个 WiFi（SSID 在电脑 WLAN 设置里看）

**2. 关闭 Windows 防火墙（或开放 8088 端口）**
   - 控制面板 → Windows Defender 防火墙 → 启用或关闭 Windows Defender 防火墙 → 关闭（测试用）
   - 或者添加入站规则：TCP 端口 8088 允许

**3. 微信开启调试模式**
   - 手机微信 → 打开小程序 → 右上角 `...` → 开发调试 → **开启调试**
   - 这会跳过 HTTPS 域名校验，允许 HTTP 请求

**4. 重启后端服务**
   - 重启 Spring Boot 项目

### 可选方案：改用热点

如果你坚持用热点（比如没有共同 WiFi），按以下步骤：

1. **Windows 设置 → 网络和 Internet → 移动热点** → 开启
2. 开启后再次运行 `ipconfig`，这时 `本地连接* 1` 或 `本地连接* 2` 会变成已连接，显示 IP（通常 `192.168.137.1`）
3. 把 `app.js` 的 `baseUrl` 改为那个热点 IP
4. 手机连电脑的热点
